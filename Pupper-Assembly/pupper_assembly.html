Pupper-Assembly

The mini-puppers are a step in a completely different direction of the Misty II bots.
The Misty bots run on tracks, the puppers are quadruped.
The Misty bot code is closed to the public with no transparency and no chance for improvement or adjustment. The mini-pupper code is open source and the development team is easily accessible via their discord.
The Misty bot uses a beginner safety first approach to robotics interaction. You can and are encouraged to experiment as much as you like with the mini-puppers.

A part of this do it yourself approach is that the puppers are shipped in parts. We needed to put them together using the TA guide, online guide, and and common sense. Although I am not typically a huge fan of putting kits together for school, putting together the puppers yourself is imperitive to future debugging. Servo fails? What is the least number of pieces that need to be removed to replace said servo? What joints control what? A screw fell out, where does it go based on its size and where screws should be. All of these questions will need to be answered at some point in any extended development.

The next part of the process is to setup the software for the pupper. As to be expected with first interactions with such sophisticated software, there is a lot to be taken in. Problems from poking around to hard is to be expected. Luckily MangDang's prebuilt iso's make the stakes low.


The pupper was the first robot that we were able to take home with us. This massively increased the amount of hours that the team could put into projects. Although constructing the pupper can be done by a single inexperienced person in ~6 distracted hours, 3 people can complete the pupper in under 2 hours.

It is my recomendation that a 3 person group splits the work up into:

  • Legs
  • Servos
  • Body and Leg Support

Once the pupper is fully constructed we needed to show that we had the pupper in a usable state. Again all of these steps proved to be invaluable for future investigation and debugging.

  1. Flash the micro-sd with the desired iso file (minipupper_ros2_Humble was the iso we use). This can be done with belena etcher.
  2. Setup wifi. You will need a keyboard initially.
    1. Use `sudo vim /etc/netplan/50-cloud-init.yaml`.
    2. Add your ssid and password.
    3. Use `sudo netplan apply`.
    4. Use `sudo reboot` to apply network changes.
  3. Ensure you have the proper repositories. If you do not, clone the repos and then run their install files.
  4. Check if the calibrate file exists. If it does not, you'll need to run the install file in the mini_pupper_bsp again. As the calibration file is setup after the dkms installations are checked, dkms cannot be installed. The process feels very janky, but it has worked to reset the calibration file on both Monday Team 1's and Wednesday Team 2's pupper.
    1. Check dkms with `dkms status`.
    2. Remove all 3 dkms types with `sudo dkms remove name/1.0 --all`
    3. Run the mini_pupper_bsp install file. It will likely fail on an os check.
    4. Run it again.
  5. Run a movement scheme. Either the PS4 Joystick movement scheme or the ros2 teleop_twist_keyboard movement scheme.
